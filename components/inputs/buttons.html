Best Practices

  • Use Clear, Action-Oriented Labels

    Labels like “Submit,” “Download,” or “Start Free Trial” are better than vague ones like “Click Here.”

  • Prioritize with Visual Hierarchy

    Use primary, secondary, and tertiary styles to guide attention. The most important action should stand out visually.

  • Provide Feedback on Interaction

    Use loading indicators, success messages, or visual changes to confirm that an action was triggered.

When Not to Use

  • Using Overly Long Button Labels

    Buttons with lengthy text can overwhelm users. Keep labels short and action-oriented.

  • Using Too Many Primary Buttons

    When everything is “primary,” nothing stands out. Limit to one main action per screen.

  • Disabling Buttons Without Explanation

    If a button is inactive, explain why (ex. “Please fill out all required fields.”)
